= Installing the Operator using the CLI

[role="_abstract"]
Install the Operator using the command line interface.

.Procedure

. Create a subscription manifest:
+
[source,yaml]
----
apiVersion: operators.coreos.com/v1alpha1
kind: Subscription
metadata:
  name: rhods-operator
  namespace: redhat-ods-operator <1>
spec:
  name: rhods-operator
  channel: __<channel>__ <2>
  source: cs-redhat-operator-index
  sourceNamespace: openshift-marketplace
  startingCSV: rhods-operator.x.y.z <3>
----
+
<1> Defines the operator namespace.
ifndef::disconnected[]
<2> Sets the update channel. You must specify a value of `fast`, `fast-x.y`, `stable`, `stable-x.y` `eus-x.y`, or `eus`.
endif::[]
ifdef::disconnected[]
<2> Sets the update channel for disconnected environments. You must specify a value of `fast`, `fast-x.y`, `stable`, `stable-x.y`, `eus-x.y`, or `eus`.
endif::[]
<3> Optional: Sets the operator version. If you do not specify a value, the subscription defaults to the latest operator version.

. Apply the subscription manifest.
