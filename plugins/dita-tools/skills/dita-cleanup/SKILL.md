# dita-cleanup

Review and integrate unhandled content in DITA files generated by dita-convert.

## Purpose

The dita-convert skill generates DITA files from AsciiDoc but may leave some elements unhandled. These unhandled elements are embedded as XML comments in the DITA output. This skill reviews the DITA files, locates these comment markers, and refactors the content into valid, semantically correct DITA XML.

## Prerequisites

- DITA files generated by dita-convert containing `<!-- UNHANDLED: ... -->` comment markers

## Workflow

1. **Read DITA files** - Open the `.dita` topic files and `.ditamap` files from dita-convert output
2. **Locate unhandled markers** - Find all `<!-- UNHANDLED: ... -->` XML comments
3. **Analyze context** - Examine the surrounding DITA structure to determine the correct placement
4. **Consult DITA DTD** - Reference the DITA 2.0 content models to ensure valid element placement
5. **Refactor content** - Convert the commented content to valid DITA XML elements
6. **Validate** - Ensure the modified DITA is valid against the DTD

## Unhandled Comment Format

The dita-convert skill embeds unhandled content as XML comments with metadata:

```xml
<!-- UNHANDLED: type=listing language=terminal parent=list_item parent_title=Verification position=1
curl -X POST -H "Content-Type: application/json" -d '{
    "model": "/models/granite-3.3-8b-instruct"
    "prompt": "What is the capital of France?",
    "max_tokens": 50
}' http://<your_server_ip>:8000/v1/completions | jq
-->
```

Comment attributes:
- `type` - Original AsciiDoc element type (listing, paragraph, admonition, ulist, etc.)
- `language` - For code blocks, the source language
- `parent` - Parent element type in the source AST
- `parent_title` - Title of the parent section (Prerequisites, Procedure, Verification, etc.)
- `position` - Position among siblings in the source

## DITA Element Placement Rules

Use the DITA 2.0 content models to determine valid placement:

### Task topics (`<task>`)

| Unhandled Type | Parent Context | Valid DITA Placement |
|----------------|----------------|----------------------|
| `listing` | Verification | `<codeblock>` inside `<postreq>` |
| `listing` | Procedure step | `<codeblock>` inside `<stepxmp>` or `<info>` |
| `listing` | Prerequisites | `<codeblock>` inside `<prereq>` |
| `paragraph` | any | `<p>` in appropriate container |
| `admonition` | Procedure step | `<note>` inside `<info>` |
| `admonition` | other | `<note>` in appropriate container |
| `ulist` | any | `<ul>` in appropriate container |
| `olist` | Procedure | `<steps>` (nested) or `<ol>` in `<info>` |

### Task element order (from DTD)

The `<taskbody>` must contain elements in this order:
1. `<prereq>` (optional)
2. `<context>` (optional)
3. `<steps>` or `<steps-unordered>` (optional)
4. `<result>` (optional)
5. `<tasktroubleshooting>` (optional)
6. `<example>` (optional, multiple allowed)
7. `<postreq>` (optional)

### Concept topics (`<concept>`)

The `<conbody>` allows most block elements in any order:
- `<p>`, `<ul>`, `<ol>`, `<dl>`, `<table>`, `<fig>`, `<codeblock>`, `<note>`, `<section>`

### Reference topics (`<reference>`)

The `<refbody>` allows:
- `<section>`, `<refsyn>`, `<table>`, `<simpletable>`, `<properties>`, `<example>`

## DITA DTD Reference

Consult the DITA 2.0 Technical Content DTD for element content models:

```
~/bin/dita-ot-4.3.1/plugins/org.oasis-open.dita.techcomm.v2_0/dtd/
├── task.dtd
├── concept.dtd
├── reference.dtd
└── technicalContent.dtd
```

Or the RNG schemas:
```
~/bin/dita-ot-4.3.1/plugins/org.oasis-open.dita.techcomm.v2_0/rng/
├── technicalContent/
│   ├── task.rng
│   ├── concept.rng
│   └── reference.rng
```

## Example Cleanup

### Before (with unhandled comment)

```xml
<task id="my-procedure">
  <title>My Procedure</title>
  <taskbody>
    <steps>
      <step>
        <cmd>Run the command</cmd>
      </step>
    </steps>
    <!-- UNHANDLED: type=listing language=terminal parent=list_item parent_title=Verification position=0
curl http://localhost:8000/v1/completions
-->
  </taskbody>
</task>
```

### After (cleaned up)

```xml
<task id="my-procedure">
  <title>My Procedure</title>
  <taskbody>
    <steps>
      <step>
        <cmd>Run the command</cmd>
      </step>
    </steps>
    <postreq>
      <p>Verify the service is running:</p>
      <codeblock outputclass="language-terminal">
curl http://localhost:8000/v1/completions
      </codeblock>
    </postreq>
  </taskbody>
</task>
```

## Validation

After cleanup, validate the DITA using DITA-OT:

```bash
~/bin/dita-ot-4.3.1/bin/dita --input my-topic.dita --format dita -o /tmp/validate
```
