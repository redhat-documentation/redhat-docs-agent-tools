# CQA 2.1 Full Checklist

Use this checklist to track assessment progress. Mark each parameter as you complete it.

## Pre-migration

- [ ] **P1** Vale DITA check (0 errors, 0 warnings)
- [ ] **P2** Assembly structure (intro + includes only)
- [ ] **P3** Content is modularized (correct prefixes)
- [ ] **P4** Modules use official templates
- [ ] **P5** All required elements present (content type, ID, title, abstract)
- [ ] **P6** Assemblies use official template
- [ ] **P7** Max 3 nesting levels
- [ ] **P8** Short descriptions are clear and actionable
- [ ] **P9** Short descriptions: 50-300 chars with `[role="_abstract"]`
- [ ] **P10** Titles support short, long, and descriptive forms
- [ ] **P11** Titles are brief, complete, and descriptive
- [ ] **P12** Prerequisites: `.Prerequisites` label, unordered list, max 10
- [ ] **P13** Grammatically correct American English
- [ ] **P14** Correct content type matches actual content
- [ ] **P15** No broken links (xrefs, includes, images)
- [ ] **P16** Redirects work
- [ ] **P17** Content interlinked within 3 clicks
- [ ] **P18** Official product names (attributes, not hardcoded)
- [ ] **P19** Tech Preview/Developer Preview disclaimers present

## Quality

- [ ] **Q1** Scannable: sentences <= 22 words avg, paragraphs 2-3 sentences
- [ ] **Q2** Clearly written and understandable
- [ ] **Q3** Simple words (no "utilize", "leverage", "in order to")
- [ ] **Q4** Readability score (11-12th grade level)
- [ ] **Q5** No fluff ("This section describes...", "as mentioned")
- [ ] **Q6** Content targets correct persona
- [ ] **Q7** Content addresses user pain points
- [ ] **Q8** Acronyms expanded at first use per guide
- [ ] **Q9** Additional resources include useful cross-site links
- [ ] **Q10** Admonitions not overused (max 3-4 per file)
- [ ] **Q11** Assembly intros target audience
- [ ] **Q12** Procedures <= 10 steps
- [ ] **Q13** Procedures include command examples
- [ ] **Q14** Optional/conditional steps formatted correctly
- [ ] **Q15** Procedures include verification steps
- [ ] **Q16** Procedures include Additional resources
- [ ] **Q17** Non-RH links acknowledged or disclaimed
- [ ] **Q18** Content follows Red Hat style guide
- [ ] **Q19** No excessive screen images
- [ ] **Q20** Appropriate conversational tone
- [ ] **Q21** Tables have captions
- [ ] **Q22** Images have meaningful alt text
- [ ] **Q23** Conscious language (no master/slave/whitelist/blacklist/dummy)
- [ ] **Q24** Links to relevant content journey
- [ ] **Q25** Content interlinked within 3 clicks

## Onboarding

- [ ] **O1** RH brand and style guidelines followed
- [ ] **O2** Copyright and legal notices present
- [ ] **O3** Official product names used
- [ ] **O4** Conscious language guidelines followed
- [ ] **O5** Tech Preview disclaimers present
- [ ] **O6** Content supported or disclaimers about support level
- [ ] **O7** Procedures tested by SME/QE
- [ ] **O8** Source files in RH prescribed format (AsciiDoc)
- [ ] **O9** Content published through Pantheon
- [ ] **O10** Content published to official Red Hat site
